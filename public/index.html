SOA Logistica / Paqueteria

Monorepo academico con tres servicios NestJS independientes: envios, paquetes y rutas. La arquitectura implementa contratos REST, Saga orquestada, Redis pub/sub y bases de datos separadas.

shipments-service

Orquestador de Saga, JWT, confirmacion de envios y publicacion de eventos.

parcels-service

Validacion de paquetes, reserva de etiqueta y compensacion.

routes-service

Cobertura, reserva de rutas y consumo del evento shipment.created.

Nota: Vercel publica esta pagina estatica del proyecto. Para ejecutar los tres servicios backend con Redis y PostgreSQL se recomienda usar Docker Compose localmente o una plataforma de contenedores como Railway, Render, Fly.io o Azure Container Apps.
